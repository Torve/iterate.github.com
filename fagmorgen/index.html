Enterprise testing with Groovy and Spock

The benefits of an automated functional testing suite are well documented, less so about how to introduce automated functional tests into a complex enterprise with distributed services and teams. There are also many documented cases of automated testing becoming a burden and not actually providing the value promised.

So how do you go about introducing automated functional testing without falling into the traps?

This miniseminar will describe the process taken at Norwegian Air Shuttle, provide an introduction to the Spock test framework and explain how combined with the language features of Groovy it is possible to write elegant functional tests that both document functionality and provide the valuable feedback required.

Intended audience: Architects, tech leads, developers, testers, everyone that writes code. May also be interesting for project managers and release managers.

Level: Intro/Beginner.

Simple breakfast and coffee will be served.
